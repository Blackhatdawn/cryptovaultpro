Your frontend (Vite on port 5173) is showing repeated errors:
  "Error: connect ECONNREFUSED 127.0.0.1:8001"

This means:
  ❌ Backend API server is NOT running on port 8001
  ❌ Frontend cannot connect to backend for API calls
  ❌ MongoDB may not be running either

SOLUTION: Start services in correct order:

  1. Start MongoDB (database)
  2. Start Backend API server (Python/FastAPI)
  3. Frontend will auto-connect (already running)

STEP 1: Verify MongoDB is Running (5 minutes)
─────────────────────────────────────────────────────────────────────────────

Choose the command for YOUR operating system:

📱 macOS:
    brew services start mongodb-community
    mongosh --eval "db.version()"

🐧 Linux:
    sudo systemctl start mongod
    mongosh --eval "db.version()"

🪟 Windows:
    net start MongoDB
    mongosh --eval "db.version()"

🐳 Docker:
    docker run -d -p 27017:27017 mongo:latest
    docker ps | grep mongodb

SUCCESS: Command shows MongoDB version (e.g., "7.0.0")

┌─ CRITICAL ──────────────────────────────────────────────────────────────────┐
│ If mongosh/mongo is not found, MongoDB is not installed.                    │
│ Visit: https://www.mongodb.com/try/download/community                       │
│ OR use Atlas cloud: https://www.mongodb.com/cloud/atlas (free tier)         │
└─────────────────────────────────────────────────────────────────────────────┘


STEP 2: Start Backend Server (5 minutes)
─────────────────────────────────────────────────────────────────────────────

Open a terminal and run ONE of these commands:

OPTION A - Using startup script (RECOMMENDED):
    macOS/Linux:
        ./start-backend.sh
    
    Windows:
        start-backend.bat

OPTION B - Manual startup:
    cd backend
    source venv/bin/activate          # macOS/Linux
    # OR
    venv\Scripts\activate             # Windows
    
    python -m uvicorn server:app --host 0.0.0.0 --port 8001 --reload

SUCCESS INDICATORS:
    ✅ See "Uvicorn running on http://0.0.0.0:8001"
    ✅ See "Application startup complete"
    ✅ curl http://localhost:8001/ping returns: {"status":"ok"...}

Keep this terminal open and RUNNING.
